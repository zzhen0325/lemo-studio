## 最终修复确认

根据您的反馈，我已彻底排查并修复了参考图残留和历史记录显示异常的问题。

### 修复核心
1.  **切断隐式回填**：在生成逻辑 (`useGenerationService.ts`) 中，彻底移除了当 UI 无图时从旧 `editConfig` 自动抓取参考图的代码。现在生成操作严格遵循当前的 `sourceImageUrls`（即 UI 上的图片）。
2.  **显示逻辑净化**：在历史记录列表 (`HistoryList.tsx`) 中，移除了当 `sourceImageUrls` 为空时回退读取 `editConfig.referenceImages` 的逻辑。现在历史记录的参考图显示完全取决于生成时刻的真实输入。
3.  **数据源一致性**：在 `useGenerationService.ts` 的 `handleGenerate` 方法中，优化了 `sourceImageUrls` 和 `localSourceIds` 的获取逻辑，优先使用传入的参数（`options`），确保批量生成和重新生成时的数据准确性。

### 预期效果
- **清空后生成**：如果您清空了 UI 上的参考图并点击生成，生成的历史记录中将**绝对不会**包含任何参考图，无论之前的编辑状态如何。
- **刷新后一致**：刷新页面后，历史记录的显示将与生成时保持完全一致，不会出现“幽灵”参考图。
- **批量/重新生成**：这些操作将正确携带当时指定的参考图，不受后续 UI 变化的影响。
