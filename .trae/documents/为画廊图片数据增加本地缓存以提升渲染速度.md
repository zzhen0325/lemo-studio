# 为画廊图片数据增加本地缓存

为了提升页面渲染速度，我们将通过 Zustand 的持久化中间件为画廊（Gallery）和历史记录（History）的元数据添加本地缓存。这将使得用户在打开页面时能立即看到之前的图片，而无需等待 API 请求完成。

## 技术方案

### 1. 引入状态持久化
在 [playground-store.ts](file:///Users/bytedance/Desktop/seeseezz/gulux/lib/store/playground-store.ts) 中引入 `zustand/middleware` 的 `persist` 插件。
- **选择性持久化**：使用 `partialize` 配置，仅对 `galleryItems` 和 `generationHistory` 进行存储，避免存储不必要的 UI 状态（如正在加载、当前页码等）。
- **存储介质**：默认使用 `localStorage`。

### 2. 优化初始化加载逻辑
修改 [GalleryView.tsx](file:///Users/bytedance/Desktop/seeseezz/gulux/components/features/playground-v2/GalleryView.tsx) 和相关组件的初始化逻辑。
- **即时渲染**：由于数据已从本地缓存恢复，页面加载时 `galleryItems` 不再为空，用户能立即看到图片。
- **后台刷新**：修改 `useEffect` 逻辑，即使缓存中有数据，在页面挂载时仍触发一次后台刷新以确保数据最新。

### 3. 数据同步与更新
- 现有的 `addGalleryItem` 和 `fetchGallery` 成功后会自动触发 Zustand 的状态更新，进而由 `persist` 插件自动同步到本地缓存中。

1. **修改 Store 配置**：

   - 实现 `partialize` 逻辑，限制持久化字段。

2. **调整 UI 初始化**：

3. **测试验证**：
   - 确认首次加载页面时，画廊内容能瞬间呈现。
   - 确认后台刷新完成后，新的生成结果能正确插入到列表中。
