# Project Structure Guide

## Root Directory Rules

Only keep project-level config and entry files in root:

- `package.json`, lock files, ts/js config
- framework config (`next.config.mjs`, `tailwind.config.ts`, etc.)
- workflow entry/config files only (no raw template JSON in root)

Do not place temporary scripts, test drafts, or generated screenshots in root.

## Recommended Folder Usage

- `app/`: Next.js App Router pages/layouts（前端渲染层）
  - `app/studio/`: studio 主业务路由（`/studio/playground`、`/studio/mapping-editor`、`/studio/gallery`、`/studio/tools`、`/studio/dataset`、`/studio/settings`）
  - `app/studio/*/_components`: route-private components（settings/tools/dataset/mapping-editor/auth）
- `server/`: Gulux backend source（唯一业务 API 实现）
- `components/`: reusable cross-route components（UI primitives, visual effects, motion/animate helpers）
- `lib/`: shared business logic, API adapters, stores
  - `lib/playground/types.ts`: route-agnostic playground shared types (avoid `lib` -> `app` reverse dependency)
- `scripts/`: maintenance scripts and one-off migration tools
- `docs/`: product/engineering docs
  - `docs/wiki/`: legacy module docs (migrated from root `wiki/`)
  - `docs/tools/`: tool development docs (for example `tldraw-dev-guide.md`)
- `tests/`: test cases
- `workflows/`: workflow templates and configs
  - `workflows/templates/flux-klein/`: Flux Klein fallback templates
- `config/legacy/`: legacy compatibility fallbacks (for example `view_comfy.json`)

## API Access Convention

- Frontend must use same-origin `/api/*` only (`getApiBase()` -> `/api` in browser).
- Next.js route handler proxy forwards `/api/:path*` to the Gulux backend (`GULUX_API_BASE`, default `http://127.0.0.1:3000/api`) and preserves streaming responses.
- Do not hardcode cross-port backend URLs in client code.
- Centralize public env reads in `lib/env/public.ts`.
- See [ENVIRONMENT.md](./ENVIRONMENT.md) for env variable contract.

## Large File Split (Ongoing)

- `app/studio/playground/_components/Dialogs/TldrawEditorView.tsx`
  - extracted widgets to `app/studio/playground/_components/Dialogs/TldrawEditorWidgets.tsx`
- `app/studio/playground/_components/containers/PlaygroundPageContainer.tsx`
  - extracted modal-state logic to `app/studio/playground/_components/containers/hooks/useResultModalState.ts`
- `lib/store/playground-store.ts`
  - extracted pure helpers to `lib/store/playground-store.helpers.ts`

## Generated Artifacts

Do not commit generated build artifacts:

- `server/output/` (Gulux build output)
- `.next/` (Next.js build output)
- `report/` (generated by `check:*` scripts)
- runtime logs and temporary output files

## Cleanup Checklist (Before Commit)

1. Ensure no temporary files were added in root.
2. Ensure build artifacts are not committed.
3. Move implementation notes to `docs/` (for example `docs/notes/`).
4. Keep tool-specific docs under `docs/tools/`.
